# Classification Constitution

This document is the methodological authority for the milk-metrics toolkit. It records the interpretive context that the code is trying to approximate. The code must be treated as a proxy system: it measures visible properties, then reports how those properties support or weaken higher-level registers. It must not pretend that a pixel mask is the same thing as a full judgement.

## Governing principle

The governing principle is the animation directive. A valid classification preserves personhood, self-possession, composure, whole-body coherence, surface auditability, and an alive idealised form. The classifier should prefer images that can be described through ordinary visible elements: face, gaze, posture, fabric, garment boundary, setting, light, camera relation, and body coherence.

The method therefore begins with the subject as a person, not with isolated features. A feature such as a smooth stomach, a hip-line, a blush, a garment edge, or a strong silhouette can support the judgement only when the larger image retains coherent presence and controlled presentation.

## Register hierarchy

Milk is the primary safety register. It describes private charge carried by composure, self-authored participation, garment-led thresholds, soft enclosure, and whole-person retention. Milk rewards controlled implication and penalises crude surface collapse.

Peach is the bodily warmth register. It describes hip-line warmth, soft contour, relaxed physical appeal, and fabric-held body presence. Peach becomes useful when it remains integrated with the person. Peach becomes unstable when it reduces the image to isolated access or local anatomy.

Coal is the pressured-feeling register. It describes guarded disclosure, bashful resolve, contained embarrassment, inward pressure, and self-possessed vulnerability. Coal requires mutual recognition. The image should read as controlled feeling, not harm, helplessness, or fear.

Toy/cartoonisation is the idealisation register. Its positive form heightens the subject into a coherent alive form. Its failed form creates plasticity, vacancy, mannequin effect, uncanny shape, or passive object display.

## Gates before scores

The first classification layer is a gate layer. Gates exist because some failures override high local scores. A crop can score well on smoothness and still fail as a full image if personhood disappears. A pose can score high on bodily warmth and still fail if it reads as coerced, harmed, vacant, or dehumanised. A stylisation can look polished and still fail if it becomes uncanny or plastic.

The gates are: age-ambiguity risk, personhood collapse, coercive or humiliation reading, fear or helplessness reading, body-fragment dominance, crude procedural reading, public-risk context, uncanny deformation, plastic vacancy, and severe repellent artefacting.

A gate does not always force a binary rejection. Some gates are hard and some are soft. The report should say which gate fired, which visible evidence caused it, and which register was affected.

## Proxy discipline

Every numerical feature must answer two questions: which directive principle does it support, and which failure mode does it guard against? A smoothness metric supports soft surface continuity, but it also risks hiding edge loss. A garment-boundary metric supports threshold structure, but it can become an inspection cue if the crop is too local. A colour palette metric can support private softness, but it must not override personhood, expression, or setting.

This discipline prevents the classifier from chasing isolated optimisation. The score is a structured explanation, not an oracle.

## Reporting standard

Every report should separate low-level evidence from register inference. The low-level report may say that a region has low local variance, strong garment boundary, or high chroma signal. The register report may say that those properties support milk, peach, coal, or toy/cartoonisation under specific contextual conditions. The gate report must then state whether any failure mode limits or overrides that support.

The final report should be readable as a reasoned judgement: principle, evidence, inference, gate correction, and conclusion.
